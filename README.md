# VibeFinder

> Find your intellectual twin. Explore a 3D map of developers shaped by what they build, play, and watch.

**Live demo:** https://vibe-finder-eta.vercel.app  
**Demo video:** https://youtu.be/vxL8L170SgI

Built for the [Qdrant Vector Space Hackathon](https://try.qdrant.tech/hackathon-vsd).

---

## What it does

VibeFinder fuses signals from three platforms into a single personality vector per user:

| Source | Signal |
|--------|--------|
| **GitHub** | Repos, stars, languages, topics |
| **Steam** | Game library, playtime, genres |
| **AniList** | Anime favourites, genres, taste tags |

Each source is embedded independently using `sentence-transformers`, then merged into one weighted vector stored in Qdrant. UMAP reduces the full collection to 3D coordinates for spatial exploration.

The result: a navigable space where proximity means similarity — and you can tune *what kind* of similarity you care about.

---

## Key interactions

**Space view** — the full UMAP map. Every node is a profile. Position encodes real semantic distance from the fused personality vector.

**Galaxy view** — select any profile and become the sun. Your closest matches orbit you, with orbit radius proportional to similarity score. Animated, bloom-lit, source-colored planets.

**Reweighting sliders** — live Qdrant re-query as you drag. Shift emphasis between GitHub, Steam, and AniList to find your coding twin, gaming twin, or anime twin from the same profile.

**Match explanations** — a one-sentence LLM-generated insight for each match, grounded in actual metadata. Not generic — specific to what you actually share.

## How it works

```
GitHub ──┐
Steam  ──┼──► sentence-transformers ──► weighted fusion ──► Qdrant
AniList ─┘                                                      │
                                                          ANN search
                                                                │
                                              UMAP 3D ◄── stored vectors
                                                │
                                          React Three Fiber
```

1. Each source is fetched via OAuth or username lookup and embedded independently
2. Source vectors are merged into a single profile vector with per-source weights stored in payload
3. Default matches use equal-weight fusion — sliders recompute the query vector on the fly without re-indexing
4. UMAP runs server-side over all stored vectors and returns 3D coordinates to the frontend
5. Match explanations are generated by Groq (`llama-3.1-8b-instant`) from user metadata

## Stack

- **Frontend:** React + Vite + React Three Fiber + `@react-three/drei` + `@react-three/postprocessing`
- **Backend:** FastAPI + Python
- **Vector DB:** Qdrant Cloud
- **Embeddings:** `sentence-transformers` (`all-MiniLM-L6-v2`)
- **Dimensionality reduction:** UMAP → 3D
- **Auth:** Supabase + Google OAuth
- **Match explanations:** Groq (`llama-3.1-8b-instant`)

## Local setup

**Backend:**
```bash
pip install -r requirements.txt
cp .env.example .env        # fill in your keys
uvicorn main:app --reload --port 8000
```

**Frontend:**
```bash
cd frontend
npm install
cp .env.example .env        # fill in Supabase + API URL
npm run dev
```

**Seed demo profiles:**
```bash
python seed.py              # GitHub profiles
python seed.py anilist      # AniList profiles
```

See `.env.example` for required environment variables.

## API endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/umap` | 3D coordinates for all profiles |
| `GET` | `/matches/{user_id}` | Default weighted matches |
| `POST` | `/matches/weighted` | Live reweighted matches |
| `POST` | `/explain` | LLM match explanation |
| `GET` | `/auth/login` | GitHub OAuth start |
| `POST` | `/steam/connect` | Connect Steam profile |
| `POST` | `/anilist/connect` | Connect AniList profile |
